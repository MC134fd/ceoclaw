Security Policy

Last updated: March 1, 2026

Buildflow takes security seriously. This policy describes our security practices, infrastructure, and responsible disclosure program.

Infrastructure security

Buildflow runs on Amazon Web Services (AWS) across two regions (us-east-1 and eu-west-1) with automatic failover. All infrastructure is managed as code and deployed via CI/CD pipelines with automated security scanning.

  • Data encrypted at rest with AES-256
  • Data encrypted in transit with TLS 1.3
  • Network access controlled via VPC and security groups
  • Regular penetration testing by independent third parties

Application security

  • Passwords hashed with bcrypt (cost factor 12)
  • JWT tokens with short expiry + refresh token rotation
  • CSRF protection on all state-changing endpoints
  • Rate limiting and abuse detection on all API endpoints
  • Dependency scanning in CI/CD pipeline (Dependabot + Snyk)

Access controls

Internal access to customer data is strictly limited on a need-to-know basis with MFA required for all staff. All access is logged and reviewed monthly. We do not access customer startup data unless explicitly requested for support purposes and authorized by you.

Compliance

  • SOC 2 Type II certified (annual audit)
  • GDPR compliant
  • CCPA compliant
  • HIPAA not applicable — Buildflow does not process health information

Incident response

In the event of a data breach, we will notify affected users within 72 hours as required by GDPR, and within the timeframes required by applicable law. We maintain a formal incident response plan that is tested quarterly.

Responsible disclosure

If you discover a security vulnerability in Buildflow, please report it to security@buildflow.app. Include a detailed description of the issue, steps to reproduce, and the potential impact. We will:

  • Acknowledge receipt within 24 hours
  • Provide an initial assessment within 72 hours
  • Work with you on a coordinated disclosure timeline
  • Credit you in our security acknowledgements (if desired)

We do not pursue legal action against researchers acting in good faith under this policy.
